Compile-Time Checks to Prevent Runtime Errors

🚫 Shape / type mismatch — DMA copy between tensors with incompatible shapes, ranks, or element types caught at compile time
🚧 Out-of-bounds access — static index/offset analysis catches buffer overruns before they become silent corruption
⚙ HW constraint validation — MMA shape/dtype must match hardware capabilities (e.g., SM_86 vs SM_90 instruction set)
🔄 Synchronization analysis — async futures that are never wait-ed are flagged at compile time, preventing deadlocks and races

In CUDA / CuTe — these are all silent bugs

  • Shape mismatches → silent data corruption
  • Out-of-bounds → intermittent wrong results
  • Wrong MMA shape → GPU hang, no diagnostic
  • Missing sync → race condition, rare crashes
"The bug that takes 5 days to find in CUDA is a compile error in CrokTile."

📦 Memory budget enforcement — cumulative dynamic allocation checked against HW limits

In CUDA — no safety net

Flexible Memory Access Primitives

📦 chunkat — tile into fixed-size chunks by index. Best for regular grids: each block processes one tile
📏 subspan/at — sub-view with explicit tile shape + tile index. Best for TMA copies where shape must be declared upfront
🔍 view/from — dynamic window with arbitrary offset. Best for irregular access: MoE segments, variable-length seqs
🔄 span_as — reshape without copy. Best for reinterpreting layouts: collapse batch dim, reshape for MMA
🔗 # operator — hierarchical index composition. Combines block + warp + thread indices into a single global index

In CUDA

  • Manual pointer arithmetic for every access pattern
  • No bounds checking — off-by-one = silent corruption
